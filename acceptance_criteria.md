# 逐阶段验收清单

> 每个阶段完成后必须通过对应验收，才能进入下一阶段。

---

## 阶段1: 预测编码时序过滤

### 1.1 因果性检查
- [ ] 线性模型 W 的拟合样本不包含目标帧 t
- [ ] 线性模型 W 的拟合样本不包含目标帧的相邻帧 (t±1)
- [ ] 预测仅使用 t-3, t-2, t-1 的历史帧
- [ ] 逐帧滑动窗口未发生索引偏移

### 1.2 阈值控制
- [ ] 二分搜索 k 值实现
- [ ] 阶段1 单独输出约占全量 35-40%
- [ ] 输出帧索引在 episode 内按时间排序

### 1.3 输出验证
- [ ] 每个 episode 至少保留 10% 帧 (防止空集)
- [ ] e_t 分布可视化确认非退化 (非全局接近零)

---

## 阶段2: RAS 关键事件检测

### 2.1 检测逻辑
- [ ] 夹爪检测使用连续梯度 + 分位阈值 (非布尔)
- [ ] 加速度使用二阶差分 + 分位阈值
- [ ] 接触窗口使用速度下降率 + 分位阈值 + 夹爪联动

### 2.2 输出验证
- [ ] 阶段2 单独输出约占全量 5-10%
- [ ] 检测到的事件帧包含真实关键动作 (人工抽检 5 帧)
- [ ] 无连续相同检测 (去重)

---

## 阶段3: Facility Location 子模优化

### 3.1 特征预处理
- [ ] 视觉特征从 512d PCA 降至 32d
- [ ] PCA 在主特征矩阵上拟合 (不使用全量测试集)
- [ ] PCA 保留方差 ≥ 85%
- [ ] 视觉和动作特征分别 L2 归一化

### 3.2 相似度核
- [ ] 使用乘积核: `S = K_v × K_a`
- [ ] 带宽 σ² 使用中位数距离估计
- [ ] 带宽在 5000 随机训练对上计算

### 3.3 Lazy Greedy
- [ ] 子模函数实现正确 (Facility Location)
- [ ] Lazy Greedy 边际增益递减正确
- [ ] 最终核心集 = 全量 10% (精确)
- [ ] 核心集内无重复索引

---

## 阶段4: 梯度精炼 (可选)

- [ ] 等量置换: 入=N, 出=N
- [ ] 迭代 2 轮
- [ ] 最终核心集仍 = 全量 10%

## Baseline 与 MLP 训练

### MLP 架构
- [ ] 输入 512d, 隐藏层 256 → 128, 输出 7d
- [ ] ReLU 激活
- [ ] 学习率 1e-3, Adam 优化器
- [ ] 训练 100 epoch, batch_size 64
- [ ] MSE Loss

### 数据划分
- [ ] 按 episode 划分 train/test (40/10)
- [ ] 核心集选择仅在 train 集
- [ ] 测试集在所有条件下固定不变

### 评估
- [ ] MSE 在测试集上计算
- [ ] 每个动作维度单独计算 MSE
- [ ] Random 基线重复 3 次取平均
